# ADR-001: eCOS 定位为外化认知操作系统

- **状态**: 已接受
- **日期**: 2026-05-13
- **决策者**: 夏同学 + Hermes

## 背景

用户拥有 SharedBrain B-OS、KOS、Minerva 三个独立系统，缺乏统一的架构层来协调它们的关系、定义共同的行为准则、保障跨 Agent 的连续性。

## 决策

将 eCOS 定位为**外化认知操作系统（External Cognitive Operating System）**，而非：
- 工具集合（不是功能堆砌）
- 另一个知识库（不替代 KOS/SharedBrain）
- 自动化平台（不替代 Hermes 的工具调用能力）

eCOS 的核心职责是：认知基础设施 + 多 Agent 宪法 + 系统演化路径。

## 后果

- 正面：有清晰的边界，避免重复建设，各系统分工明确
- 风险：架构层如果过重，会成为负担而非加速器
- 缓解：eCOS 自身保持轻量（文档 + 规范 + Workflow），不运行业务逻辑
