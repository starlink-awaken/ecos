# ADR-003: 建立 Agent 委员会作为多 Agent 协作机制

- **状态**: 已接受（章程 v0.1.0-draft）
- **日期**: 2026-05-13
- **决策者**: 夏同学 + Hermes

## 背景

单一 Agent 存在视角单一、确认偏误、专业覆盖不全等固有缺陷。需要一套机制支持多 Agent 协作，同时保证决策的正确性和可持久性。

## 决策

建立 **Agent 委员会**机制：
- 四个常设角色：CHAIR（主持）、EXEC（执行）、AUDIT（审查）、SCRIBE（记录）
- 四个专业角色：RESEARCH、KOS、CRITIC、PLANNER（按需召集）
- 三种工作流程：标准流程（六阶段）、快速流程（三角模式）、紧急流程
- 法定人数：≥3 个角色参与，或 2/3 多数赞成
- Hermes 作为主 Agent（Master），拥有召集权和 L2 配置权，但重大决策不能独断

委员会通过 Workflow 实施，结果通过 SSB 总线通信，所有决策通过 SCRIBE 归档。

## 后果

- 正面：多角度决策，有批评机制（CRITIC 角色），可审计
- 风险：委员会流程增加决策时间
- 缓解：低风险任务走快速流程（三角模式，≤10分钟）
- 待验证：首次实际 Workflow 运行后需修订章程
