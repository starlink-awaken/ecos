# eCOS 六层架构详细设计

> 对应 GENOME.md 第六部分的展开文档

---

## 架构全图

```
┌─────────────────────────────────────────────────────────┐
│  Layer 6: 反馈层 (Feedback)                              │
│  宪法执行器 | 稳态监控 | 偏差检测 | 纠偏触发              │
├─────────────────────────────────────────────────────────┤
│  Layer 5: 行动层 (Action)                                │
│  Hermes工具集 | 外部API | Workflow执行器 | MCP调用        │
├─────────────────────────────────────────────────────────┤
│  Layer 4: 智能层 (Intelligence)                          │
│  Agent委员会 | 推理引擎 | 规划器 | SSB路由               │
├─────────────────────────────────────────────────────────┤
│  Layer 3: 持久层 (Persistence)                           │
│  KOS(知识索引) | SharedBrain(B-OS) | 失败案例库           │
├─────────────────────────────────────────────────────────┤
│  Layer 2: 感知层 (Perception)                            │
│  Capture → Filter → Structure → Contextualize → Integrate│
├─────────────────────────────────────────────────────────┤
│  Layer 1: 宪法层 (Constitution)                          │
│  GENOME.md | L0不变量 | L1宪法 | L2政策                  │
└─────────────────────────────────────────────────────────┘

               ↕ SSB 共享语义总线（贯穿所有层）
```

---

## Layer 1: 宪法层

**职责：** 定义系统的边界和不变量，所有其他层必须在此约束内运行。

**组件：**
- GENOME.md：系统宪法文档
- L0 绝对不变量（永不可修改）
- L1 宪法层（RFC + 人类确认变更）
- L2 政策层（Master Agent 可配置）

**接口：**
- 只读接口（所有层可读）
- 写入接口（仅 RFC 流程 + 人类确认）

---

## Layer 2: 感知层

**职责：** 系统与外界信息的唯一合法接口，负责降熵。

**五阶管道：**
```
Capture       → 原始信息捕获（截图/文本/API/用户输入）
    ↓
Filter        → 噪音过滤，相关性判断
    ↓
Structure     → 结构化（Markdown/JSON/Schema）
    ↓
Contextualize → 语义标注，关联已有知识
    ↓
Integrate     → 写入持久层（KOS/SharedBrain）
```

**关键原则：** 未经感知层处理的原始信息不直接写入持久层。

---

## Layer 3: 持久层

**职责：** 系统的长期记忆，知识的持久化存储。

**三个存储系统：**

| 系统 | 职责 | 技术栈 |
|------|------|--------|
| KOS | 6域文档索引，~23万文档，全文检索 | FTS5 + Neo4j + LanceDB |
| SharedBrain B-OS | 仿生OS，BOS-URI路由，跨系统信息存储 | Python + bun + MCP |
| 失败案例库 | 失败经验持久化，防止重蹈覆辙 | Markdown文件 + 未来可索引 |

---

## Layer 4: 智能层

**职责：** 推理、规划、多 Agent 协作决策。

**核心组件：**
- Agent 委员会（见 CHARTER.md）
- SSB 路由器（事件分发）
- 规划器（任务分解，PLANNER 角色）
- 推理引擎（当前 = LLM，未来可扩展）

---

## Layer 5: 行动层

**职责：** 将智能层的决策转化为实际操作。

**执行能力：**
- Hermes 22+ 工具（terminal/browser/file等）
- MCP 工具调用（KOS/Minerva/SharedBrain）
- 外部 API（GitHub/Discord/Email等）
- Workflow 执行器（cron任务，委员会决议执行）

---

## Layer 6: 反馈层

**职责：** 监控系统是否偏离三类稳态，触发纠偏。

**三类稳态监控：**

```
宪法稳态检查：行为是否违反 L0/L1 不变量？
  → 违反：立即停止 + 人类通知

知识稳态检查：知识库是否与现实一致？
  → 偏离：触发感知层重新采集

目标稳态检查：行动是否与用户意图对齐？
  → 偏离 > 30%：重新召集委员会
```

**宪法执行器：** 反馈层的核心组件，每次行动层执行后运行，验证结果符合宪法约束。

---

## SSB 共享语义总线

SSB 贯穿所有层，是 eCOS 的神经系统：

```
信息流向：
感知层输入 →[SSB]→ 持久层存储
智能层决策 →[SSB]→ 行动层执行
行动层结果 →[SSB]→ 反馈层验证
反馈层纠偏 →[SSB]→ 智能层更新
```

**设计原则：**
- 异步消息总线，不阻塞 Agent 执行
- 所有消息持久化（可回溯）
- 信息素场语义：Agent 读取 SSB 历史可感知系统状态
