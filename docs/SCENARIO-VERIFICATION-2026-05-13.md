## 新发现问题

### N1: Minerva 研究超时 (MCP 120s 限制)

```
触发条件: 复杂查询或长管道(deep_read阶段)
发生频率: ~25% (6次调用中2次超时)
影响: L0研究不可靠，需重试或缩短查询
根因: KOS MCP 默认 timeout=120s，Minerva L0管道需47-120s
```

**缓解方案：**
- 短期: Agent 在超时后自动用更短查询重试
- 中期: KOS MCP 配置中增加 timeout 到 180s

### N2: gongwen 域 PDF 全文搜索受限

```
触发条件: 搜索"信息化建设"在gongwen命中文档但标题不含关键词
影响: PDF/Word文档的正文内容可能未建立全文索引
      搜索结果依赖标题和snippet而非正文
根因: 需确认KOS索引器对PDF/Word的提取策略
```
