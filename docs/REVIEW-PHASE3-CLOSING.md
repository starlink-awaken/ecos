## 五、你的两个问题

### Q: 为什么微信回复专业丰富, CLI 回复简短?

这不是 bug。是不同的会话上下文和模型导致的：

**微信 (Weixin) 会话:**
- 使用 deepseek-v4-flash 模型
- 独立会话, 需要从 STATE+HANDOFF 冷启动
- 每次回复需要加载完整上下文 → 输出更详细
- 跨会话的 HANDOFF 机制强制了信息密度

**CLI 会话:**
- 使用 deepseek-v4-pro 模型
- 长会话, 上下文持续累积
- 模型倾向于简洁 (上下文窗口压力)
- 更"对话式", 省略已知信息

**HANDOFF 机制的效果:**
微信 Agent 每次启动必须读 STATE+HANDOFF → 输出包含完整状态摘要。CLI Agent 在同一会话中已经知道了上下文 → 省略冗余。这不是质量差异, 是上下文密度差异。如果 CLI 也是新会话冷启动, 回复也会同样详细。
