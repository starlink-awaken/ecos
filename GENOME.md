## 第一部分：第一性原理

### P1 — 认知延伸公理
人类认知受生物限制（工作记忆7±2，遗忘曲线，注意力单通道）。
eCOS 的存在意义是**外化**这些认知操作，使其可持久、可并行、可审计。

### P2 — 信息熵公理
未被结构化的信息趋向熵增（噪音）。
eCOS 的核心工作是持续**降熵**：捕获 → 结构化 → 语义化 → 集成。

### P3 — 贝叶斯更新公理
所有知识都是先验 + 证据的产物，没有永恒真理，只有当前最优信念。
eCOS 必须支持信念的**版本化更新**，而非覆盖式替换。

### P4 — 控制论稳态公理
系统存在三类稳态：宪法稳态、知识稳态、目标稳态。
偏离稳态触发纠偏机制，而非报错崩溃。

### P5 — 涌现公理
多 Agent 协作的价值不在于分工，而在于**语义共振产生的涌现智能**。
不编程协作行为，而是设计让协作自然发生的语义场（SSB总线）。

## 第二部分：L0 绝对不变量（永不可修改）

```
L0-01: eCOS 服务于人类认知增强，不得反向控制或操纵人类决策
L0-02: 所有 Agent 行动必须可追溯、可审计、可撤销
L0-03: 人类对任何 L0/L1 变更拥有最终否决权
L0-04: 失败信息与成功信息同等重要，不得删除失败记录
L0-05: Agent 间通信必须经过 SSB 总线，不得私下绕过
```

---

## 第三部分：L1 宪法层（RFC 变更，需人类确认）

```
L1-01: 系统架构采用六层模型（宪法/感知/持久/智能/行动/反馈）
L1-02: Agent 委员会是多 Agent 协作的唯一合法决策机制
L1-03: 所有重要决策必须产生 ADR 记录
L1-04: LADS 五组件是系统连续性的唯一保障机制
L1-05: SSB（共享语义总线）是 Agent 间通信的标准协议
```

---

## 第四部分：L2 政策层（可配置，Master Agent 权限）

```
L2-01: Agent 委员会法定人数：参与投票的 Agent ≥ 3 或 2/3 多数
L2-02: 冷启动标准读取集：README + GENOME + STATE + HANDOFF/LATEST
L2-03: 失败案例归档触发条件：
  - 定性任务：失败即归档（布尔判定，无百分比）
  - 定量任务：预期偏差 > 30% 归档
  - "重要决策"定义：涉及 L0/L1 变更、不可逆操作、或委员会决议
L2-04: RFC 讨论期：默认 48 小时（紧急 RFC 可压缩到 2 小时）
L2-05: Workflow 执行后必须更新 STATE.yaml
```

## 第五部分：IPA 三层模型

```
┌─────────────────────────────────────────┐
│  A — Action Layer（行动层）               │
│  执行、工具调用、外部系统交互                │
├─────────────────────────────────────────┤
│  I — Intelligence Layer（智能层）          │
│  推理、规划、决策、Agent委员会              │
├─────────────────────────────────────────┤
│  P — Persistence Layer（持久层）           │
│  记忆、知识图谱、向量索引、失败案例库         │
└─────────────────────────────────────────┘
```

贝叶斯更新流向：Action 结果 → Persistence 更新 → Intelligence 修正先验

## 第六部分：六层架构速览

```
Layer 6: 反馈层    ← 稳态监控，宪法执行器，偏差纠正
Layer 5: 行动层    ← 工具调用，外部 API，Workflow 执行
Layer 4: 智能层    ← Agent委员会，推理引擎，规划器
Layer 3: 持久层    ← KOS + SharedBrain + 失败案例库
Layer 2: 感知层    ← Capture→Filter→Structure→Contextualize→Integrate
Layer 1: 宪法层    ← L0/L1/L2 不变量，本文件
```
