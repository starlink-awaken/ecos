# Agent 委员会 — Phase 1 三角模式

> Phase 1 实现方案：Hermes 角色切换模拟三角委员会
> 版本: v0.1.0
> 创建: 2026-05-13
> 原因: 逻辑推演发现完整8角色委员会在Phase1无法实施
> 关联: CHARTER.md（Phase2 完整版）

---

## 为什么需要三角模式

CHARTER.md 定义了 8 角色委员会，但 Phase 1 的现实是：
- Hermes 是唯一的持久 Agent
- delegate_task 只支持 3 个同步子 Agent（且是叶子节点，完成后销毁）
- 没有独立 Agent 进程基础设施

**三角模式**是 Phase 1 的务实方案：Hermes 在同一会话内轮流扮演 CHAIR、EXEC、AUDIT 三个角色，通过明确的角色切换提示实现多角度思考。

---

## 三角模式 vs 完整委员会

| 维度 | 三角模式 (Phase 1) | 完整委员会 (Phase 2+) |
|------|-------------------|----------------------|
| 实施方式 | Hermes 角色切换 | delegate_task 3+ Agent 并行 |
| 决策质量 | 单模型多角度（受模型偏见限制） | 多 Agent 独立判断 |
| 独立性 | 模拟的（同模型扮演不同角色） | 真实的（不同 Agent 实例） |
| 适用任务 | 中等风险及以下 | 高风险/关键决策 |
| 耗时 | ~5-10 分钟 | ~15-30 分钟 |

---

## 三角角色定义（Phase 1 缩减版）

| 角色 | 在 Phase 1 的职责 |
|------|------------------|
| CHAIR | 主持会议结构，确保议程完整，不跳过任何阶段 |
| EXEC | 提出执行方案，分解步骤，评估可行性 |
| AUDIT | 独立审查，指出遗漏的风险和 L0/L1 违规 |

> Phase 1 不设 CRITIC（由 AUDIT 兼任）、SCRIBE（由 Hermes 本身兼任）、RESEARCH/KOS/PLANNER（按需调用工具即可）。

## 三角模式流程

```
Phase 0: 角色声明（30秒）
  → Hermes 宣布三角模式启动
  → 声明本轮三个角色由同一 Agent 扮演
  → 读取 STATE.yaml + HANDOFF/LATEST.md

Phase 1: CHAIR 模式（1分钟）
  → 明确议题和决策边界
  → 列出需要达成的具体产出
  → 声明"现在切换到 EXEC 角色"

Phase 2: EXEC 模式（2-3分钟）
  → 提出执行方案（具体步骤、工具、时间估算）
  → 列出方案假设和前提条件
  → 声明"现在切换到 AUDIT 角色"

Phase 3: AUDIT 模式（2-3分钟）
  → 逐条审查 EXEC 方案
  → 对照 GENOME.md 检查 L0/L1 合规性
  → 指出被遗漏的风险
  → 提出修改建议
  → 声明"审查完成，交回 CHAIR"

Phase 4: 决策（1分钟）
  → CHAIR 综合 EXEC 方案 + AUDIT 反馈
  → 形成最终决策
  → 如风险等级 HIGH/CRITICAL → 必须人类确认
  → 如风险等级 MED 且 AUDIT 不反对 → 可执行

Phase 5: 归档
  → 更新 STATE.yaml
  → 写入 HANDOFF/LATEST.md
  → 如有重要决策 → 写入 ADR
  → 如有失败 → 写入 FAILURES/
```

---

## 角色切换规则（防污染）

切换角色时必须：
1. 明确声明"现在切换到 [角色] 身份"
2. 以前一个角色的产出作为输入（不重新生成）
3. AUDIT 不能修改 EXEC 的方案，只能提出审查意见
4. 最终由 CHAIR（而非 EXEC）拍板

---

## 何时触发三角模式

**必须触发：**
- 涉及不可逆操作（三级操作清单中的任何一项）
- 涉及 GENOME.md 的 L1/L2 修改提案
- 新 Workflow 首次运行
- 错误率/异常率显著升高

**建议触发：**
- 复杂多步任务（≥5 个步骤）
- 涉及多个外部系统协调
- 用户明确要求"多角度评估"

**不需要触发（直行）：**
- 简单信息查询
- 文件读写（非破坏性）
- 工具单步调用

---

## 升级到完整委员会的条件

当以下条件同时满足时，从三角模式升级到完整委员会：
1. delegate_task 支持 ≥4 个并发 Agent（或独立 Agent 进程可用）
2. CHAIR 判定任务风险为 HIGH 或 CRITICAL
3. 时间允许（完整委员会需 ~30分钟）
